项目简介：

注意力监测功能主要通过传感器和算法实时监测驾驶员的注意力状态，防止因分心或疲劳导致事故。核心逻辑分为数据采集、状态分析和预警触发三个部分。

数据采集通常依赖车内摄像头、方向盘传感器或红外传感器。摄像头捕捉驾驶员的面部特征（如眼睑闭合度、视线方向），方向盘传感器检测握力变化，红外传感器辅助夜间监测。

状态分析环节采用计算机视觉和机器学习算法。面部特征数据通过卷积神经网络（CNN）提取关键点，结合时序模型（如LSTM）判断注意力持续性。

一句话：YOLOv8n-Pose = YOLOv8 最轻量人体关键点模型，1 个模型同时输出：人体框 + COCO17 个人体关键点 (x,y, 可见度)，主打极速推理、嵌入式 / 车载部署

1.Backbone 骨干（CSPDarknet+SPPF，大量残差块）
5 级 CSP 层（残差 + 跳跃连接，解决深层梯度消失，深浅特征留存）
SPPF 池化：多尺度全局特征融合
输出 3 种尺度特征：80×80(小目标)、40×40、20×20(大目标)
驾驶员场景：80×80 浅层抓眼睛、鼻尖细微关键点；深层识别人体躯干
2.Neck 颈部 PAN-FPN 双向特征金字塔
自上而下 + 自下而上双向融合：
FPN：深层高级语义往下传
PAN：浅层纹理细节往上拼
关键点检测必备：兼顾人脸小关键点 + 整个人体
3.Head 头部：双分支解耦头（Pose 核心改动）
普通 YOLO 只有检测分支 (框 + 类别)；Pose 改成并行双分支：
检测分支：识别人体、输出人体 bounding box
关键点分支（新增）：每人预测COCO 标准 17 个关键点
每个关键点输出：[x坐标,y坐标,visible可见置信度]（遮挡时 visible 偏低）
17 个关键点（你做驾驶员注意力只用前 6 个头部点）
1. 鼻尖 2. 左眼 3. 右眼 4. 左耳 5. 右耳 6. 左肩… 剩余躯干四肢
疲劳检测只用：眼、鼻关键点 → 算闭眼、打哈欠、低头角度
